# Environment Setup

## Prerequisites

- Flutter (version pinned in `.fvmrc` — use [FVM](https://fvm.app/) or install that version manually)
- A Firebase project with Authentication (Google) and Firestore enabled

## Flutter app

The app needs two secret files that are gitignored:

### 1. `lib/firebase_options.dart`

Generated by `flutterfire configure`, or copy from an existing team member. This file wires the app to your Firebase project. CI injects it from the `FIREBASE_OPTIONS_DART` GitHub secret.

### 2. `.env`

Used by the Node upload scripts in `scripts/` (not by the Flutter app itself). Copy from the template:

```bash
cp .env.example .env
```

Fill in the values from your Firebase project:

| Variable | Where to find it |
|---|---|
| `GOOGLE_WEB_CLIENT_ID` | Firebase Console → Authentication → Sign-in method → Google → Web SDK configuration |
| `FIREBASE_PROJECT_ID`, `FIREBASE_API_KEY`, etc. | Firebase Console → Project Settings → General → Your apps → Config |
| `FIREBASE_PRIVATE_KEY`, `FIREBASE_CLIENT_EMAIL`, etc. | Firebase Console → Project Settings → Service accounts → Generate new private key |

CI injects `.env` from the `DOT_ENV` GitHub secret.

## Running

```bash
flutter pub get
flutter run -d macos    # or -d windows / -d linux / -d chrome
flutter test            # run tests
flutter analyze         # lint check
```

## CI / Deployment

All workflows live in `.github/workflows/` and read the Flutter version from `.fvmrc`:

- **Deploy Preview** (`deploy-preview.yml`) — triggers on push to `main`, deploys to Firebase Hosting preview channel.
- **Deploy Production** (`deploy-production.yml`) — manual trigger, deploys to the `live` channel.
- **Release Desktop** (`release-desktop.yml`) — manual trigger with version input, builds macOS/Windows/Linux and creates a draft GitHub Release.

Both deploy workflows need these GitHub secrets:
- `FIREBASE_OPTIONS_DART` — contents of `lib/firebase_options.dart`
- `DOT_ENV` — contents of `.env`
- `FIREBASE_SERVICE_ACCOUNT` — Firebase hosting service account JSON

## Troubleshooting

| Problem | Fix |
|---|---|
| `firebase_options.dart not found` | Run `flutterfire configure` or get the file from a team member |
| Google Sign-In fails on web | Add your domain (including `localhost`) to Firebase Console → Authentication → Settings → Authorized domains |
| Google Sign-In fails on desktop | Ensure the OAuth client ID in `firebase_options.dart` matches the one in Firebase Console |
| Firestore permission denied | Check `firestore.rules` — ensure the authenticated user's UID matches the document path |
| `flutter pub get` fails | Verify your Flutter version matches `.fvmrc` (`flutter --version`) |
| Upload scripts fail with auth errors | Check that `.env` has valid `FIREBASE_PRIVATE_KEY` and `FIREBASE_CLIENT_EMAIL` |
